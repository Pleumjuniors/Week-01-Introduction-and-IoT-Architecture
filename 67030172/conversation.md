คำถาม:  
“อธิบายว่า Internet of Things (IoT) คืออะไร และยกตัวอย่างระบบ IoT ที่พบในชีวิตประจำวัน พร้อมอธิบายว่าทำไมระบบนั้นจึงถือเป็น IoT”

คำตอบ:
Internet of Things (IoT) คือ การที่อุปกรณ์ต่าง ๆ เชื่อมต่อกับอินเทอร์เน็ตเพื่อส่งหรือรับข้อมูล และสามารถทำงานหรือควบคุมได้โดยอัตโนมัติหรือผ่านสมาร์ตโฟน ตัวอย่าง เช่น กล้องวงจรปิด Wi-Fi ที่สามารถดูภาพสดผ่านมือถือได้จากทุกที่ และแจ้งเตือนเมื่อมีการตรวจจับความเคลื่อนไหว ระบบนี้ถือเป็น IoT เพราะกล้องเชื่อมต่ออินเทอร์เน็ต ส่งข้อมูลมายังผู้ใช้ และสามารถสั่งงานผ่านแอปพลิเคชันได้จากระยะไกล

Prompt หมวด B — ข้อที่ 1: Internet Layer IoT Network Technologies

คำถาม:  
“อธิบายเทคโนโลยีการสื่อสารในชั้น Internet Layer ที่ใช้ในระบบ IoT เช่น IPv6, 6LoWPAN, RPL  
พร้อมอธิบายว่าทำไมเทคโนโลยีเหล่านี้จึงเหมาะกับอุปกรณ์ IoT ที่มีข้อจำกัดด้านพลังงานและแบนด์วิธ”

คำตอบ:
เทคโนโลยีการสื่อสารใน Internet Layer ที่ใช้ในระบบ IoT ได้แก่

IPv6 ใช้สำหรับกำหนดหมายเลข IP ให้กับอุปกรณ์ ทำให้สามารถรองรับอุปกรณ์จำนวนมากได้
6LoWPAN ช่วยบีบอัดข้อมูลของ IPv6 ให้มีขนาดเล็กลง เพื่อให้ส่งผ่านเครือข่ายที่มีแบนด์วิธต่ำได้
RPL เป็นโปรโตคอลสำหรับเลือกเส้นทางการส่งข้อมูลที่เหมาะสม ช่วยลดการใช้พลังงานและเพิ่มประสิทธิภาพในการสื่อสาร

เทคโนโลยีเหล่านี้เหมาะกับอุปกรณ์ IoT เพราะช่วยลดปริมาณข้อมูลที่ต้องส่ง ใช้พลังงานน้อย รองรับอุปกรณ์จำนวนมาก และทำให้การสื่อสารมีประสิทธิภาพ แม้จะใช้เครือข่ายที่มีแบนด์วิธจำกัดหรืออุปกรณ์ที่มีทรัพยากรน้อยก็ตาม

TA ตอบ :
1) ความถูกต้องของเนื้อหา (40 คะแนน)

คะแนน: 39/40

✅ IPv6

อธิบายได้ถูกต้องว่า

ใช้กำหนดหมายเลข IP ให้กับอุปกรณ์
รองรับอุปกรณ์จำนวนมาก

ถือว่าถูกต้องตามหลักการ เนื่องจาก IPv6 มีพื้นที่ Address ขนาดใหญ่ เหมาะกับการเชื่อมต่ออุปกรณ์ IoT จำนวนมหาศาล

หากเพิ่มว่า "IPv6 มี Address 128 บิต จึงรองรับอุปกรณ์ได้จำนวนมาก" จะสมบูรณ์ยิ่งขึ้น

✅ 6LoWPAN

อธิบายได้ถูกต้องว่า

ช่วยบีบอัด Header ของ IPv6
ทำให้ส่งข้อมูลผ่านเครือข่ายที่มีแบนด์วิธต่ำได้

แม้ว่าจะใช้คำว่า "บีบอัดข้อมูลของ IPv6" ซึ่งเข้าใจได้ แต่ตามหลักวิชาการจะตรงกว่าหากระบุว่า บีบอัดส่วน Header ของ IPv6 ไม่ใช่ข้อมูลทั้งหมด

เป็นเพียงรายละเอียดเล็กน้อย จึงไม่กระทบภาพรวมมาก

เลือกเส้นทางที่เหมาะสม
ลดการใช้พลังงาน

ถือว่าตรงตามบทเรียน

Prompt หมวด C — ข้อที่ 1: IoT‑System‑Challenges

คำถาม:  
“อธิบายความท้าทายหลักของการสร้างระบบ IoT เช่น ระยะทางในการสื่อสาร, Bandwidth, Power usage, การเชื่อมต่อเครือข่าย, การทำงานร่วมกับระบบอื่น และความปลอดภัย  
พร้อมยกตัวอย่างสถานการณ์จริงที่แสดงให้เห็นว่าความท้าทายเหล่านี้ส่งผลต่อการออกแบบระบบ IoT อย่างไร”

คำตอบ:
การสร้างระบบ IoT มีความท้าทายหลายด้าน ได้แก่

ระยะทางในการสื่อสาร อุปกรณ์ต้องส่งข้อมูลได้ในระยะที่เหมาะสม
Bandwidth หากมีข้อมูลจำนวนมาก อาจทำให้การส่งข้อมูลช้าหรือเกิดความล่าช้า
Power usage อุปกรณ์ IoT ส่วนใหญ่ใช้แบตเตอรี่ จึงต้องประหยัดพลังงาน
การเชื่อมต่อเครือข่าย ต้องมีสัญญาณอินเทอร์เน็ตที่เสถียรเพื่อให้ระบบทำงานได้ต่อเนื่อง
การทำงานร่วมกับระบบอื่น อุปกรณ์จากหลายผู้ผลิตต้องสามารถสื่อสารกันได้
ความปลอดภัย ต้องป้องกันข้อมูลและป้องกันการเข้าถึงจากผู้ไม่หวังดี

ตัวอย่าง: ระบบตรวจวัดอุณหภูมิในฟาร์ม เซนเซอร์ต้องส่งข้อมูลจากพื้นที่ห่างไกล จึงต้องเลือกเครือข่ายที่ส่งได้ไกลและใช้พลังงานต่ำ หากอินเทอร์เน็ตล่มจะส่งข้อมูลไม่ได้ และต้องมีระบบรักษาความปลอดภัยเพื่อป้องกันการเข้าถึงข้อมูลหรือสั่งงานระบบโดยไม่ได้รับอนุญาต
